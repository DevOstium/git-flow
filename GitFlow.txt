Git Flow

   É uma forma de estruturar o versionamento dos projetos com convenções.
   
//----------------------------------------------------------------------   
	
	Sempre Branch: Master e Developer
	
	Crio as novas branches na Devolper

	
	A branch Master      = Produção
	A branch Developer   = A tudo que está sendo desenvolvido
	
	
	Publicar tag:  $ git push origin numero_tag. exemplo: 2019.01

	$ git flow hotfix publish 2019.01
	
	$ git flow hotfix publish 2019.01.01
	
	$ git flow hotfix finish 2019.01.01  --   apos finalizar a branch o git exclui a branch
	
	$ git push origin developer
	
	$ git push origin master
